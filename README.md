---

## Database Components

### **1. Operational OLTP Schema**
Includes core business entities:

- **Buildings**  
- **Apartments**  
- **Clients**  
- **Staff**  
- **Leases**  
- **Maintenance Requests**  
- **Inspections**

This schema enforces:

- Primary & foreign keys  
- CHECK constraints  
- Cascading rules  
- Consistent relationships  
- Clean normalized design  

### **2. Analytical Reporting Layer**
A simplified star style model was created for:

- Occupancy metrics  
- Maintenance performance  
- Utility & cost analysis  
- Turnaround time calculations

## üìä Business Insights Enabled

This SQL system allows property managers to answer questions such as:

- What is the occupancy rate by building, apartment type, or time period?  
- How long do maintenance tasks take to complete on average?  
- Which buildings generate the most maintenance workload?  
- Which clients or unit types have the highest turnover?  
- Which cost centers contribute most to operational expenses?  

These insights support operations, resource allocation, and financial decision making.
